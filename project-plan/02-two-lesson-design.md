# 两节课课程设计

## 第一节：把记忆讲完整

### 课堂目标

第一节不是只完成一张手写故事卡，而是建立家庭数字记忆档案 V1。

目标链条：

```text
真实故事采集 -> 结构化整理 -> AI 辅助润色 -> 长者核验 -> 建立数字档案 V1
```

### 建议流程

| 时间 | 内容 | 产出 |
| --- | --- | --- |
| 0-10 分钟 | 示范故事与最终成果说明 | 理解“档案包”成果 |
| 10-15 分钟 | 教授“记忆五要素” | 采访任务卡 |
| 15-38 分钟 | 老少结对采访、录音、拍照 | 原始故事素材 |
| 38-48 分钟 | AI 整理示范与小组体验 | 故事初稿 |
| 48-55 分钟 | 长者核对事实、修改表达 | 确认稿 |
| 55-60 分钟 | 扫描二维码预览、总结下节课 | 数字档案 V1 入口 |

### 课堂重点

第一节课要把“故事采集”做扎实。AI 整理环节只需要让参与者看到：同样的口述素材，可以被整理成标题、正文、金句和待确认信息。

不建议在第一节安排复杂 AI 视频生成。视频生成耗时不稳定，也容易让课堂重心偏离真实讲述。

### 第一节课后团队工作

团队课后完成：

1. 整理录音和采访卡；
2. 统一修订故事文字；
3. 标注不确定信息；
4. 将照片和故事录入家庭数据表；
5. 生成每个家庭的数字页面；
6. 生成二维码预览卡；
7. 准备第二节课扫码复看。

## 第二节：让记忆有声音、有画面

### 课堂目标

第二节不换故事，而是继续完善第一节课的同一份档案，形成家庭数字记忆档案 V2。

目标链条：

```text
扫码复看 -> 核对故事 -> 录制声音 -> 可选 AI 影像 -> 最终授权 -> 数字档案 V2
```

### 建议流程

| 时间 | 内容 | 产出 |
| --- | --- | --- |
| 0-8 分钟 | 扫码查看上节课故事页面 | 检查文字与图片 |
| 8-18 分钟 | 讲解记忆数字化五步法 | 理解 AI 的辅助作用 |
| 18-35 分钟 | 录制长者声音或老少访谈 | 声音明信片素材 |
| 35-48 分钟 | 制作音频、字幕或 AI 影像 | 数字作品 |
| 48-55 分钟 | 家庭选择版本、确认展示权限 | 最终确认 |
| 55-60 分钟 | 扫码观看、家庭分享 | 数字档案 V2 |

### 第二节必做成果

每组家庭至少补充一段声音：

1. 长者朗读整理后的故事；
2. 儿童提问、长者回答；
3. 长者说一句最想留给家人的话。

声音比 AI 视频更真实，也更适合所有家庭完成。

### 第二节可选成果

AI 影像作为自愿升级或精选成果：

1. 老照片轻微动态化；
2. 老物件图文短片；
3. 怀旧场景生成图；
4. 配合声音制作短视频。

建议只选择素材质量较好、授权清楚、现场时间允许的家庭完成，不要求全员当堂生成。

## 两节课之间的衔接话术

第一节结尾可以这样说：

> 今天我们先把家里的记忆讲清楚、写下来。下一节课，我们会继续使用同一个二维码，把这段故事变成可以听见的声音明信片，也可以根据授权尝试制作一段 AI 记忆影像。

第二节开头可以这样说：

> 上次我们已经完成了数字档案的第一版。今天不是重新做一个故事，而是把同一份档案继续完善，让它从“能读”变成“能听、能看、能扫码保存”。
