# 项目总方案：家庭数字记忆档案

## 1. 项目定位

“岁月留痕”不是单纯的 AI 工具体验课，也不是只做一张手写故事卡的手工活动。更适合的定位是：

> 以老物件、老照片和家庭口述为入口，帮助长者与儿童共同完成一份可保存、可展示、可更新的家庭数字记忆档案。

课程中的 AI 不是主角，而是辅助工具。核心价值仍然来自长者的真实经历、儿童的提问、家庭成员之间的交流，以及团队对记忆内容的尊重整理。

## 2. 成果名称

建议统一命名为：

> 《家庭数字记忆档案包》

每组家庭最终拿到的不只是课堂作业，而是一份完整的小型家庭记忆档案。它既能作为社区展陈成果，也能作为家庭内部保存和转发的纪念材料。

## 3. 成果结构

基础成果建议所有家庭都完成：

1. 正式排版的老物件故事卡；
2. 专属二维码数字展签；
3. 独立家庭数字记忆页面；
4. 30-60 秒声音明信片。

升级成果根据时间、授权和素材质量选择完成：

1. 10-30 秒 AI 记忆影像；
2. 图文短片；
3. 可用于社区展览的大屏展示版。

## 4. 两课闭环

第一堂课的重点是“把故事讲完整”。课堂产出是故事素材、照片、长者原话、手写故事卡草稿和数字档案 V1 的入口。

第二堂课的重点是“让记忆有声音、有画面”。课堂继续使用第一堂课的同一份档案，补充长者声音、儿童访谈、AI 影像和最终展示授权。

最终闭环如下：

```text
第一堂课：真实故事采集 -> AI 辅助整理 -> 本人核验 -> 数字档案 V1
第二堂课：扫码复看 -> 录制声音 -> 可选 AI 影像 -> 数字档案 V2
```

## 5. 推荐技术路线

首轮建议采用：

```text
VS Code + Codex 辅助开发
+ Python 批量生成网页和二维码
+ HTML/CSS 展示
+ GitHub Pages 或静态托管上线
```

这样可以避免第一阶段就陷入小程序注册、服务器、数据库、用户权限和审核流程。对于 10-20 个家庭的首轮实践，静态网页足够支撑交付和展示。

## 6. 核心判断

这个方案可行的关键不在于技术多复杂，而在于流程是否稳定：

1. 每个家庭有固定编号和固定二维码；
2. 第一堂课后页面可显示照片和故事；
3. 第二堂课后同一个页面补充声音和影像；
4. 二维码不更换，只更新页面内容；
5. 所有公开展示内容都经过授权和本人核验。

首轮验收标准建议设为：

> 同一个二维码，第一堂课后能看到照片和故事，第二堂课后能继续看到声音和可选影像。
