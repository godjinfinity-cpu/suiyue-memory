# 家庭档案内容与数据规范

## 1. 家庭档案页面内容

每个家庭对应一个独立页面，建议页面包含以下模块：

1. 老物件或老照片图片；
2. 故事标题；
3. 年代、人物、地点；
4. 150-250 字故事正文；
5. 一句长者原话；
6. 讲述人与儿童共创人；
7. 声音明信片；
8. AI 记忆影像，可选；
9. AI 辅助整理说明；
10. 展示权限。

页面需要优先适配手机端，因为居民大概率通过微信扫码访问。

## 2. 故事卡正式版内容

课堂手写卡只是草稿。正式故事卡建议包含：

```text
故事标题：
老物件或老照片：
年代：
相关人物：
地点：
故事正文：
长者原话：
讲述人：
儿童共创人：
AI 辅助整理说明：
展示权限：
活动日期：
```

正文建议控制在 150-250 字，保留口述温度，不追求过度文学化。

## 4. 编号与二维码

建议每组家庭使用固定编号：

```text
F001
F002
F003
```

二维码不要直接指向姓名页面，而是指向带随机后缀的页面：

```text
/f/f001-a7k9m2/
```

这样可以降低被随意猜到的概率。但需要明确：随机地址不等于密码保护。如果内容真正私密，不应该放在公开静态网站上。

## 5. 展示权限

建议设置三档：

```text
private    仅家庭查看
community  社区内部展示
public     可公开展示
```

第一版如果使用 GitHub Pages，需要特别注意：GitHub Pages 本质上是公开网页。标记为 `private` 的内容不应上传到公开网站。

## 6. AI 整理提示词

用于故事整理：

```text
请根据以下真实口述素材，整理一篇适合社区数字记忆展展示的故事。

物件或照片：【】
大致年代：【】
相关人物：【】
发生的事情：【】
长者原话：【】
想表达的情感：【】

要求：
1. 不虚构原故事中没有的信息；
2. 保留长者朴素、自然的表达；
3. 字数控制在 150-250 字；
4. 输出故事标题、正文、最动人的一句话；
5. 对不确定的信息标注“需要进一步确认”。
```

用于影像生成：

```text
年代【】+ 地点【】+ 人物关系【】+ 核心物件【】+ 动作【】+ 情感【】+ 画面风格【温暖、真实、怀旧、不过度夸张】
```

## 7. 内容审核原则

每条故事最终上线前至少检查：

1. 是否存在 AI 编造；
2. 是否有明显事实错误；
3. 是否泄露真实住址、电话、身份证号等敏感信息；
4. 是否涉及未授权人物肖像；
5. 是否涉及家庭矛盾、疾病、财产等敏感内容；
6. 是否由长者或家属确认展示。
